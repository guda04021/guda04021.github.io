做链路聚合可以提高我们网络的可靠性，同时能够增加带宽，也可以为我们提供备份

# 网络可靠性需求

生成树协议，避免环路产生

可要能够提供冗余备份，能够避免环路的同时又充分利用带宽该怎么做呢？？

# 链路聚合技术原理与配置

链路聚合分为两类：

  1. 手动链路聚合
  2. LACP 动态协商：设备和设备之间相互交互协议报文，在链路发生故障后，它可以从非活动接口中挑选其一出来作为我们的一个备用端口

虽然设备与设备之间有四根链路存在，但其实只有一根链路在工作，当链路发生故障时会启用下一根备份链路；但是当正常工作时，剩余三根链路处于空闲状态，造成带宽浪费。这也是生成树协议的缺点所在

那么如何解决这个问题呢？

这时就需要用到我们的链路聚合技术了

转发数据实际运用时存在两种负载分担的方式：基于数据包和基于数据流；
前者可以真正实现我们链路上的负载分担（即每一根链路转发的流量是一样的，但存在一个乱序的问题）而后者可以解决乱序的问题

# 堆叠 / 集群概述

链路聚合是在链路层面实现高可靠性，若是在设备层面实现高可靠性我们可以采取堆叠 / 集群的技术

堆叠 / 集群可以避免单点故障，可以实现跨设备的链路聚合，利用率 100％

  1. 扩充端口
  2. 提高带宽
